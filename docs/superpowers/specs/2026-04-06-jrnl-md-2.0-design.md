# jrnl-md 2.0 Design Spec

## Vision

jrnl-md 2.0 is a clean-sheet rebuild. The project moves from "100%
workalike of jrnl" to "inspired by jrnl" — a focused CLI tool for
managing a directory of markdown day files. The unix ecosystem handles
everything else.

**Design principle: the day is the atom.** One markdown file per
calendar day. No per-entry structure within a day. All operations
(write, list, filter, edit) work at the day level.

## What's Gone

Removed from 1.x with no replacement:

- Starred entries
- Encryption (age)
- Export formats (JSON, XML, YAML, markdown, text)
- `--import`
- `--delete` (interactive per-entry deletion)
- `--change-time`
- `--config-override`
- `--diagnostic`, `--debug`
- `--format fancy`/`boxed`, `display_format` config
- Per-entry concept (`Entry` type, `FormatEntries`, `ParseMultiDay`,
  filtered-edit serialization, entry-level tag extraction)
- Golden test suite against jrnl
- Compat test framing

## What's New or Changed

- Subcommand CLI (`write`, `edit`, `list`, `tags`) instead of
  flag-on-root
- `Day` as the core type (replaces `Entry`)
- Configurable timestamps (on by default, optional)
- `default_list_count` config key
- Cleaner day file format (no brackets around times)
- `-year`, `-month`, `-day`, `-today-in-history` filters
- `--all` flag on `list`

## Day File Format

Files stored at `YYYY/MM/DD.md` under the journal root.

### With timestamps enabled (default)

```markdown
# 2026-04-06 Sunday

## 09:00 AM

Went for a morning run. Feeling good about the week ahead.

## 02:30 PM

Met with @sarah about the project timeline. Need to follow
up on the budget question.
```

### With timestamps disabled

```markdown
# 2026-04-06 Sunday

Went for a morning run. Feeling good about the week ahead.

Second write of the day, just appended with a blank line separator.
```

### Rules

- Line 1 is always `# YYYY-MM-DD Weekday`
- Timestamp headings are `## time` with no brackets (configurable
  format via `time_format`)
- Consecutive writes are separated by a blank line
- When timestamps are on, each write gets a `## time` heading,
  a blank line, then the body
- Tags are `@word` in body text, no special handling on write
- File extension is always `.md`

## Package Layout

```
cmd/jrnl-md/
  main.go              # Cobra root + subcommand wiring
  write.go             # write subcommand
  edit.go              # edit subcommand
  list.go              # list subcommand
  tags.go              # tags subcommand
  completion.go        # shell completion subcommand

internal/
  journal/
    day.go             # Day type, parse/format day files
    store.go           # Store: list, load, save, delete day files
    filter.go          # Filter struct, date/tag matching at day level
  config/
    config.go          # Config struct, TOML load/save, defaults
  dateparse/
    dateparse.go       # Parse natural-language and explicit dates
  display/
    display.go         # Terminal width, text wrapping, tag highlighting
  editor/
    editor.go          # Launch external editor, cursor positioning
```

Atomic file writes (temp + fsync + rename) are inlined in `store.go`.
No `prompt`, `export`, or `crypto` packages.

## Core Types

### Day

```go
type Day struct {
    Date time.Time
    Body string
}
```

`Body` is everything after the `# date` heading. Tags are extracted
from `Body` on demand when filtering — no stored `Tags` field.

### Store

```go
type Store struct {
    root    string
    dateFmt string
    timeFmt string
    tagSyms string
}
```

Methods:

- `Load(date time.Time) (*Day, error)` — read a single day file
- `Save(day Day) error` — write a day file (create or overwrite)
- `Delete(date time.Time) error` — remove a day file, clean empty
  parent dirs
- `List(f Filter) ([]Day, error)` — list days matching filter, uses
  directory structure to skip irrelevant years/months
- `Append(body string) error` — append to today's file; creates file
  with day heading if needed; adds timestamp heading if configured
- `Tags(f Filter) (map[string]int, error)` — tag frequencies across
  matching days

### Filter

```go
type Filter struct {
    Tags           []string
    NotTags        []string
    AndTags        bool
    Start          *time.Time
    End            *time.Time
    Contains       string
    N              int
    Year           int
    Month          int
    DayOfMonth     int
    TodayInHistory bool
}
```

Filters operate at the day level. Tag matching scans the day's body
for `@word` patterns using configured tag symbols.

## Config

File: `~/.config/jrnl-md/config.toml` (XDG-aware, overridable via
`--config-file`). Auto-created with defaults on first run.

```toml
[general]
editor = ""
timestamps = true
linewrap = 79
default_list_count = 10

[format]
time = "03:04 PM"
date = "2006-01-02"
tag_symbols = "@"

[colors]
date = "none"
body = "none"
tags = "none"

[journals.default]
path = "~/Documents/Journal/"
```

### Notes

- `timestamps`: controls whether `## time` headings are added on
  write. Default true.
- `default_list_count`: how many days `list` shows with no `-N` flag.
  Default 10.
- `editor`: falls back to `$VISUAL` then `$EDITOR`.
- `default_hour`/`default_minute` from 1.x are removed — writes
  always use the current time.
- `colors`: `black|red|green|yellow|blue|magenta|cyan|white|none`.
- Tag highlighting is always on (no `highlight` toggle).
- `file_extension` is always `md` (not configurable).
- `[journals.default]` is the only journal in 2.0. The schema shape
  supports future multi-journal without config migration.

## CLI Subcommands

### `jrnl-md write <text>`

Append text to today's day file. Creates the file with a day heading
if it doesn't exist. Adds a `## time` heading if timestamps are
enabled. Always targets today — no date prefixes.

### `jrnl-md edit [--on <date>]`

Open a single day file in the configured editor. Defaults to today.
`--on` selects a specific date. If the day file doesn't exist, create
it with the day heading and open it for writing. No range filters —
edit operates on exactly one day.

### `jrnl-md list [flags] [@tag...]`

Display days matching filters. Defaults to the last
`default_list_count` days.

Flags:

| Flag | Description |
|---|---|
| `-N` | Last N days (overrides default_list_count) |
| `--all` | Show all days |
| `--short` | Date + first content line, truncated |
| `--from <date>` | Days from date |
| `--to <date>` | Days up to date |
| `--on <date>` | Single day |
| `-year <year>` | All days in a year |
| `-month <month>` | All days in a month across years |
| `-day <N>` | All days on a day-of-month across months |
| `-today-in-history` | Today's date in prior years |
| `--and` | Require all specified tags |
| `--not <tag>` | Exclude days with tag |
| `--contains <text>` | Days containing text (case-insensitive) |

Positional `@tag` arguments filter to days containing those tags.

### `jrnl-md tags [flags]`

List all tags with frequency counts, sorted descending. Accepts the
same date filter flags as `list` (`--from`, `--to`, `--on`, `-year`,
`-month`, `-day`, `-today-in-history`).

### `jrnl-md completion {bash,zsh,fish,powershell}`

Shell completion scripts via Cobra.

### `jrnl-md --version`

Print version and exit.

### `jrnl-md` (no subcommand)

Show help.

## Display

### Full list output

```
# 2026-04-06 Sunday

Went for a morning run. Feeling good about the week ahead.

Met with @sarah about the project timeline. Need to follow
up on the budget question.

# 2026-04-05 Saturday

Finished reading that book on distributed systems.
```

Days separated by a blank line. Body text wrapped at `linewrap`
width. Tags highlighted in color. Date heading colored per config.
Output looks like the files because it essentially is the files.

### Short list output

```
2026-04-06  Went for a morning run. Feeling good about the week …
2026-04-05  Finished reading that book on distributed systems.
2026-04-03  Quarterly Review
```

Date from the file path, two spaces, first non-blank content line
after the day heading. If the first content line is a timestamp
heading (matches configured time format), skip it and use the next
non-blank line. Markdown heading markers (`##`) are stripped. Line
truncated to terminal width with `…`.

### Tags output

```
@work: 12
@sarah: 5
@reading: 3
```

Frequency-sorted, descending.

## Testing Strategy

Clean-sheet test suite. No golden tests against jrnl, no compat
framing.

### Unit tests

Each `internal/` package has tests against real temp directories:

- `journal/` — parse day files, store operations (load, save, append,
  delete, list), filter matching, tag extraction
- `config/` — load, save, defaults, auto-create
- `dateparse/` — explicit formats, natural language, edge cases
- `display/` — wrapping, truncation, tag highlighting, short format

### E2E tests

Build the binary, run subcommands against temp journals:

- `write_test.go` — append, timestamps on/off, creates day file and
  heading
- `edit_test.go` — opens correct day file, `--on` flag
- `list_test.go` — default count, `-N`, `--all`, `--short`, date
  filters, tag filters, `--contains`, combined filters
- `tags_test.go` — frequency sort, with date filters
- `config_test.go` — auto-creates on first run, `--config-file` flag

No mocks. Real files, real binary, real temp dirs.
